Issues

MAD-X issues

The issue tracker allows to follow-up defect reports and feature requests efficiently.

Each bug report or feature request sent to mad-usr, mad-dev or mad support will be loaded into the system, get a ticket number, assigned to a module keeper or a MAD team member (see contributors), and the reporter will be added to the notification list for follow-up. Users can also setup issues by themselves directly on the system.

Report should always come with a clear description of the problem as well as a self contained minimalistic script example using small lattices that exhibits the described unexpected behavior with the last release (see releases), otherwise the bug report might not be processed.
