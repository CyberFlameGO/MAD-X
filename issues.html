MAD-X trac system

The trac system is a powerful tool with a web interface, which allows to follow-up defect reports and feature requests efficiently. Each bug report or feature request sent to mad-usr, mad-dev or mad support will be loaded into the tracker, get a ticket number, assigned to a module keeper or a MAD team member (see contributors), and the ticket URL will be sent back to the reporter for follow-up. The latter will be informed about the progress of its tickets until their closure. To follow-up the activity of all tickets, one can subscribe to the e-group mad-trc (see support).

The following web links provide direct access to all:

  • tickets
  • active tickets
  • my active tickets (if logged in Trac)
For customized tickets list, use the "view tickets" button. For more information on tickets, see the Trac Ticket System help.

MAD-X issues

Non authenticated users on the trac system can send their reports or requests to mad support. Each report should come with a clear description of the problem as well as a self contained minimalistic script example using small lattices that exhibits the described unexpected behavior with the last development release (see releases), otherwise the bug report might not be processed.

Every authenticated users on the trac system can create and modify tickets; with care since tickets cannot be deleted and history cannot be modified. New tickets should follow the same conventions as existing tickets. In particular, all tickets comment should start with the report date in the format year.month.day (e.g. 2012.10.24), even if it is the same as the ticket creation date. A component must be selected and the email address of the reporter must be valid. In case of problem please send the report (comments and attachements) to mad support, we will load the report into the trac system for you.

Some advanced features of the trac system can be used within tickets. For examples tickets can refer to code revision, and code commits can trigger actions on tickets (e.g. close it) using special Trac links tags wherever Wiki formatting applies.
