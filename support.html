Support

The MAD team can be contacted by email at mad support (see also bottom left corner). This task is usually very complex and finding effective answer can take a significant amount of time, so we will do our best to answer accurately within a reasonable amount of time. Questions and bug reports should be supported by self contained minimalistic examples that exhibit the unexpected behavior with the last development release, provided that it is not already recorded in the issue tracker (see issues).

  • Archive of mad support (restricted to MAD team).

The MAD community can be contacted through e-groups like mad-usr (see hereafter). All the e-groups described in this page require to be a member for posting and receiving emails. Membership subscription can be directly performed by CERN users through the e-groups system, or by contacting the MAD team.

Public annoucement

The e-group mad-pub (wide audience, very low bandwidth) is recommended for any official announcement like project updates or public events related to MAD topics. This e-group is where new development and production releases are announced (see releases), and includes automatically all members of the mad-usr and be-dep-abp e-groups.

  • Subcribe/unsubscribe to mad-pub.
  • Archive of mad-pub.
  • Post to mad-pub.

Users information

The e-group mad-usr (medium audience, low bandwidth) is recommended for MAD users to publish bug reports, feature requests, documentation improvement or simply share ideas and information. This e-group automatically includes all members of the mad-dev and be-dep-abp-hss e-groups.

  • Subcribe/unsubscribe to mad-usr.
  • Archive of mad-usr.
  • Post to mad-usr.

Developers information

The e-group mad-dev (small audience, low-to-medium bandwidth) is recommended for MAD developers to publish bug reports, feature requests or share ideas and information. Information on the faulty source code would help a lot to identify and correct the bug, and ultimately corrections or patches would be a must!

  • Subcribe/unsubscribe to mad-dev.
  • Archive of mad-dev.
  • Post to mad-dev.

Bug & request information

The e-group mad-trc (small audience, medium bandwidth) is recommended for MAD users who want to be informed about bugs and requests activity (i.e. ticket notification). This e-group is not archived since it is not supposed to be used for posting emails, please consider to use mad-usr instead.

  • Subcribe/unsubscribe to mad-trc.

Source code information

The e-group mad-src (small audience, medium-to-high bandwidth) is recommended for MAD developers who want to be informed about source code activity (i.e. commit notification). This e-group is not archived since it is not supposed to be used for posting emails, please consider to use mad-dev instead.

  • Subcribe/unsubscribe to mad-src.
