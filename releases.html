Releases

MAD-X binaries are released for the Linux, Mac OS X and Windows platforms for 32 bit and 64 bit architectures (since release 5.00.09). All production and development releases are all equally tested on all platforms (since release 5.01.05) and therefore the preferred version is the last 64 bit development release for your platform. The source release of MAD-X includes code written in C, C++, Fortan90 and Fortran77.

To quickly know the latest release numbers, see Changes.

To quickly know a sketchy list of changes between releases, see Changelog.

MAD-X development releases

MAD-X development releases are numbered m.pp.dd where dd ≠ 00 is the development release number following the production release number pp. Development releases are delivered about once every four months in average and include last changes and bugs correction.

The last development release can be downloaded from the development releases repository. The parent directory link gives access to all the previous releases.

MAD-X production releases

MAD-X Production releases are numbered m.pp.00 where m is the major release number and pp the production release number. Production releases are delivered about once per year.

The last production release can be downloaded from the production releases repository. The parent directory link gives access to all the previous releases.

MAD-X source releases

MAD-X source code for the last production or development release can be downloaded from the respective production or development release repository in the form of a compressed tarball. The parent directory link gives access to all the previous releases.

MAD-X source code can be browsed or downloaded from the svn web interface (trunk) for the most recent version and from the svn web interface (tags) for specific releases.

MAD-X source code can be browsed or downloaded from the guest access (trunk) for the most recent version and from the guest access (tags) for specific releases.

MAD-X source code can be checkout using SVN: 

svn co svn+ssh://svn.cern.ch/reps/madx/trunk/madX
for read-write restricted access and: 
svn co http://svn.cern.ch/guest/madx/trunk/madX
read-only guest access.

The current source code checkout (nightly updated) can be accessed from the source code repository.

MAD-X previous releases

MAD-X previous releases can be downloaded from the releases repository. Note: users on OS X 10.5 (Leopard) need madx release 5.00.19 or later (users on OS X 10.6 or later are not concerned).

MAD-X previous releases (including binaries, libraries and source code) can be accessed on lxplus.cern.ch from the directory ~mad/madx/releases (see MAD-X on lxplus.cern.ch).

MAD-X old releases (releases pre-5.00.00) can be accessed on lxplus.cern.ch from the directory ~mad/madx/releases/old.

MAD-X local install

The following instructions assume that you downloaded the 64 bit development release from the development release repository.

On Linux (all distributions)

  • In a terminal, execute the commands 
    mv madx-linux64 madx
chmod u+x madx
./madx

On MacOSX (starting from Leopard)

  • In a terminal, execute the commands 
    mv madx-macosx64 madx
chmod u+x madx
./madx

On Windows (starting from Windows XP)

  • In a terminal, execute the commands 
    move madx-win64.exe madx.exe
./madx.exe

The same instructions can be followed for installing 32 bit releases or the numdiff tool required for running MAD-X tests. It is necessary to add the architecture type after the name (e.g. madx64) if you plan to install and use 32 bit and 64 bit releases together.

MAD-X on lxplus.cern.ch

MAD-X releases can be run on lxplus.cern.ch (AFS) by specifying the absolute path 

/afs/cern.ch/user/m/mad/bin

(shortcut ~mad/bin) in the command or by adding it to your PATH environment variable.

The following binaries and aliases are provided: 

madx          alias for last production release (madx64)
madx32        last 32 bit production  release
madx64        last 64 bit production  release

madx_dev      alias for last development release (madx_dev64)
madx_dev32    last 32 bit development release
madx_dev64    last 64 bit development release

MAD-X previous releases can be run by specifying the version number m.pp.dd (e.g. 5.00.12) after the releases path 

~mad/bin/rel/m.pp.dd/madx-linux32
~mad/bin/rel/m.pp.dd/madx-linux64

The alternate paths for MAD versions are deprecated but should still work (unmodified). 

/afs/cern.ch/group/si/slap/bin
/afs/cern.ch/group/si/slap/script
/afs/cern.ch/eng/sl/MAD-X/pro/bin
/afs/cern.ch/eng/sl/MAD-X/pro/bindev
/afs/cern.ch/eng/sl/MAD-X/dev/bin

For interactive use of MAD-X on lxplus, it is convenient to wrap madx into the rlwrap application (see Resources).

libMADX on lxplus.cern.ch

MAD-X is also provided as a standalone static and dynamic libraries on lxplus.cern.ch (AFS) at the absolute path 

/afs/cern.ch/user/m/mad/lib

(shortcut ~mad/lib), which can be added to the search paths of your project linker.

The following librarries and aliases are provided (same nomenclature as for MAD-X): 

libmadx.a         alias for last production release (libmadx64)
libmadx32.a       last 32 bit production release
libmadx64.a       last 64 bit production release

libmadx_dev.a     alias for last development release (libmadx_dev64)
libmadx_dev32.a   last 32 bit development release
libmadx_dev64.a   last 64 bit development release
For dynamic (i.e. shared) versions of the libraries, replace the .a by .so.

libMADX previous releases (starting from 5.00.17) can be found by specifying the version number m.pp.dd (e.g. 5.00.17) after the releases path 

~mad/lib/rel/m.pp.dd/libmadx-linux32
~mad/lib/rel/m.pp.dd/libmadx-linux64

Old MADs on lxplus.cern.ch

The old versions of MAD, namely mad8 and mad9, are available in the same directory path 

/afs/cern.ch/user/m/mad/bin

mad8 requires a dictionnary file named dict that can be copied from the file mad8.dict from this repository.

All the MAD versions can be launched through the front-end script mad which supports various options.

The unsupported old releases of MAD can be downloaded from the

  • mad8 releases repository (version 8.51/15).
  • mad9 releases repository (version 9.5/7).
